# Security Policy

We take the security of Nongorr and our customers' data seriously.

## Reporting a vulnerability

If you believe you have found a security vulnerability, please report it to us
**privately** — do **not** open a public GitHub issue or disclose it publicly
before it is resolved.

**Email:** nongorr.anika@gmail.com

Please include as much of the following as you can:

- A description of the vulnerability and its potential impact
- Steps to reproduce it (a proof-of-concept, if possible)
- Any relevant URLs, logs, or screenshots

We will acknowledge your report as soon as we can, keep you informed while we
investigate, and let you know when the issue is resolved. We ask that you give
us a reasonable amount of time to fix the issue before any public disclosure.

## Scope

This policy covers the Nongorr storefront and admin application
(<https://nongorr.com>) and this repository.

## Please do not

- Access, modify, or delete data that does not belong to you
- Run denial-of-service tests, send spam, or degrade the service for others
- Use automated scanners that generate high-volume traffic
- Publicly disclose an issue before we have resolved it

Acting in good faith under this policy, we will not pursue action against you
for your research. Thank you for helping keep Nongorr and our customers safe.
